# Changelog

本文件记录服务运营平台原型的功能版本变化。版本号遵循语义化版本：

- `v0.x.0`：一个可演示的功能迭代版本
- `v0.x.y`：同一版本内的修复或小调整
- `v1.0.0`：五个核心模块形成完整可演示版本

## Unreleased - v0.2.0

### Planned

- 增加技术支持工单 L1 / L2 / L3 三环处理流程。
- 增加分层技术支持人员组：L1 一线、L2 二线专家、L3 高级专家/研发支持。
- 增加分层 SLA：L1 2小时、L2 24小时、L3 72小时。
- 工单默认进入 L1 处理，L1 无法处理时手动升级 L2，L2 无法处理时手动升级 L3。
- 工单列表增加 Tab 筛选：全部、我的待处理、已解决待确认、已关闭、已超时。
- 工单详情移除“回复处理意见”功能。
- 给出解决方案后进入“已解决待确认”。
- 已关闭的服务站提单触发服务站满意度评分。
- 已关闭的 400 工单触发客服代录满意度评分。

## v0.1.0 - 2026-05-11

### Added

- 完成技术支持工单基础闭环原型。
- 支持服务站提单、400 工单、车联网告警三类来源。
- 支持工单列表筛选、详情查看、新建、编辑、保存草稿、提交工单。
- 支持受理、分派、要求补充材料、模拟服务站补充、给出解决方案、退回处理中、关闭工单。
- 支持本地测试数据和 `localStorage` 持久化。
- 支持导出当前工单数据 JSON。

### Notes

- 本版本为纯前端静态原型，暂未接入后端服务。
- 工单数据为演示数据，刷新后保留在浏览器本地存储中。
