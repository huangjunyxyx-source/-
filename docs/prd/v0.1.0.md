# PRD v0.1.0 - 技术支持工单基础闭环

## 目标

完成主机厂服务运营平台中“技术支持工单”模块的第一版可操作闭环，用于演示内部技术支持/服务运营团队如何接收、处理、补充、解决和关闭工单。

## 范围

- 技术支持工单列表与筛选。
- 技术支持工单详情。
- 技术支持工单创建与编辑。
- 工单处理流转。
- 本地测试数据与浏览器本地持久化。

## 服务问题来源

- 经销商售后服务站提单。
- 400 工单。
- 车联网告警。

## 核心能力

- 用户可以查看技术支持工单列表，并按关键词、来源、状态、优先级、处理人、车型、是否关联告警、SLA 筛选。
- 用户可以查看工单详情，包括基础信息、服务站与车辆信息、故障与诉求信息、关联信息、处理记录。
- 用户可以新建工单、保存草稿、提交工单、编辑工单。
- 用户可以受理、分派、要求补充材料、模拟服务站补充材料、给出解决方案、退回处理中、关闭工单。
- 工单操作写入处理记录。
- 页面使用 `localStorage` 保留操作后的测试数据。

## 非目标

- 不接入真实后端。
- 不实现真实服务站端入口。
- 不实现事件单、问题单、知识库工单、知识门户的完整模块。
- 不实现真实 IM 群、真实附件上传、真实权限系统。

## 验收标准

- 打开页面后能看到带测试数据的技术支持工单列表。
- 筛选条件可正常影响列表结果。
- 点击工单可进入详情页。
- 新建、编辑、保存草稿、提交工单可正常改变数据。
- 处理动作可正常改变状态并生成处理记录。
- 刷新浏览器后，操作数据不会丢失。
